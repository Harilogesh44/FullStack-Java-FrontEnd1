SwiftCheckout Overview

Your unified retail management control panel

Security Architecture

All protected endpoints require a valid HS256-signed JWT Bearer token. Tokens are issued at login and expire after 24 hours. Admin routes require ROLE_ADMIN, while order views are restricted to ROLE_CUSTOMER.

System Architecture

Built on a layered Spring Boot architecture: REST Controllers delegate to Service classes, which interact with JPA Repositories backed by MySQL. DTOs decouple the API surface from internal Hibernate entities.

Role-Based Access

ROLE_ADMIN — full access: product management, inventory adjustments, all dashboard sections.
ROLE_CUSTOMER — order viewing, checkout, payment processing.

Database & Seeding

MySQL on jdbc:mysql://localhost:3306/swiftcheckout with Hibernate DDL auto-update. DataSeeder pre-populates admin, cashier, and 3–5 sample products with inventory on first startup.
